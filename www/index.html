What is Kretase?

Your own Pterodactyl,
built with modern tools

Kretase is a self-hosted game server management panel — think Pterodactyl, but built on a modern TypeScript stack you can actually read and modify.

You install it on your own Linux server in one command. It sets up Nginx, PostgreSQL, SSL certificates, and a web panel automatically. From there, you can deploy Minecraft servers (and more game types soon) from a clean browser interface — without touching the command line again.

The Wings daemon runs on each game node, giving you Docker-isolated, resource-limited containers per server. Every server gets its own CPU/RAM limit, its own file system, and its own console — visible live in the browser.

  • 100% open source — fork it, modify it, own it
  • One command installs everything end-to-end
  • Docker isolation — no server can crash another
  • Real-time console, live stats, file manager in browser
  • JWT auth + 2FA + role-based access control
  • Runs on Ubuntu 20/22/24 and Debian 11/12

Features

Everything you need to run
game servers at scale

Built for server owners who want power without complexity. No PHP, no bloat — just a clean modern stack that does exactly what you need.

Real-Time Console

WebSocket-powered server console with live output streaming. Send commands, monitor logs, and track events — all in real time.

[12:04:31][Server/INFO] Done (2.8s)! For help, type "help"
[12:04:45][Server/INFO] Player_1 joined the game
› say Welcome to the server!
[12:04:46][Server/INFO] [Server] Welcome!

Resource Monitoring

Live CPU, RAM, and disk graphs updated in real time. Know exactly what your servers are doing at all times.

Multi-Node Support

Connect unlimited Wings daemon nodes. Spread servers across multiple machines from one central panel.

Auto Backups

Schedule automatic server backups and restore them with a single click. Never lose your world data again.

Secure by Default

JWT authentication, 2FA support, role-based access control, and auto-SSL via Let's Encrypt.

Team Management

Invite teammates with granular permission levels. Give admins, mods, and players exactly the access they need.

Docker Isolation

Each server runs in its own Docker container. Full process isolation — no server can affect another.

File Manager

Browse, edit, upload, and delete server files directly from the browser. Full SFTP access also available.

Scheduled Tasks

Automate server restarts, commands, and backups with cron-based scheduling. Set it and forget it.

Sub-User Access

Grant teammates granular access to specific servers. Control who can use the console, files, power actions, and more.

Server Notes

Keep private notes on each server — configs, IPs, credentials, todos. Always visible right where you need them.

Server Templates

Launch new servers in seconds with pre-configured templates — Survival, Minigame, Creative, Modded, Bedrock, and more.

Plugin Marketplace

Search and install plugins directly to your server from the panel — no manual file transfers needed.

Mod Manager

Same one-click experience for Fabric mods, with automatic loader and Minecraft version matching.

Version Manager

Browse and switch Paper versions with changelogs, downgrade warnings, and an optional backup before installing.

World Manager

Switch, back up, and export your worlds — or browse and install premade worlds like castles and mansions from CurseForge.

AI Tools

Generate MOTDs and server logos for free with a built-in algorithm, or plug in your own OpenAI, Gemini, or Anthropic key for AI-powered results.

Installation

Up and running
in under 5 minutes

One command sets up Nginx, PostgreSQL, SSL certificates, and the panel. Another command on each game server installs the Wings daemon.

01

Get a server

Any Ubuntu 20.04/22.04/24.04 or Debian 11/12 VPS with root access. Minimum 1 GB RAM.

02

Run the installer

One command installs everything — Node.js, Nginx, PostgreSQL, SSL, and the panel itself.

03

Install Wings on nodes

On each game server, run the Wings installer. Connect it to your panel via the admin UI.

04

Create servers

Pick a game, allocate resources, and deploy. Your server starts in seconds.

root@server:~
# Install the panel
$ bash <(curl -fsSL https://raw.githubusercontent.com/mwlih28/mc-manage-panel/main/scripts/install-panel.sh)

  ✓ Panel running at https://panel.yourdomain.com
 
# On each game server node
$ bash <(curl -fsSL https://raw.githubusercontent.com/mwlih28/mc-manage-panel/main/scripts/install-wings.sh)
  ✓ Docker installed
  ✓ Wings daemon running
  ✓ Node connected to panel

Security

Security isn't a checkbox —
it's built in

No paid audit badge to point at yet — we're a young project. What we do have is real, verifiable engineering: automated scanning on every commit, and a public track record you can check yourself.

Encrypted Passwords

Every password and API key secret is hashed with bcrypt (cost factor 12) — never stored, logged, or transmitted in plain text.

Two-Factor Authentication

Optional TOTP-based 2FA on every account, checked with any standard authenticator app.

Scoped API Keys

Admin-only, least-privilege API keys with per-key permissions and expiry — never one all-powerful token.

Hardened by Default

Content-Security-Policy, HSTS, and rate-limited login/2FA endpoints ship out of the box — no config needed.

Automated Vulnerability Scanning

CodeQL static analysis and Dependabot dependency alerts run on every commit — not just before a release.

Public Disclosure Process

Found a bug? Report it privately via GitHub Security Advisories — see SECURITY.md for the full process.
